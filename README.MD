# Defect 缺陷机器人

Defect 是一个高可配置性，高兼容性，优秀harness，节省资源的无头通用agent

Defect 本仓库不提供任何ui实现，你可以通过ACP协议，在您喜欢的任何前端中来调用它

## 为什么不要TUI
大家对tui的吹捧其实是一种船货崇拜。agent本身是一个服务一般的角色，服务和它的界面应当是解耦的，但是不知道为什么，现在每一个claude code like的应用，都会带一个跟自己的服务绑定的tui。
我并不觉得tui很好用，反而它还限制了富文本，多媒体，复杂编辑能力。如果tui真的很好用，现在主宰编辑器领域的就该是neovim了！

既然zed提出了ACP协议，我想我应该去拥抱ACP。agent开发者就可以更好专注于agent本身，图形界面自然会有人去做。拥抱标准是好的，协议为我们提供了一个“窄腰”，把多对多的复杂兼容变成了多对一。

## 本项目的指导性纲领
1. **开放**，我们开放每一层可配置的配置，让Defect做到高度的可定制
2. **紧凑**，我们争取用最小的代价，实现最全面的功能
